# 🛵 外卖配送平台 — 三人分工方案（比例 4:3:3）

---

## 系统交互总览

该图展示了"外卖配送平台"与外部实体的整体交互关系。系统作为一个整体数据处理单元，与四个核心外部实体及一个支付渠道进行数据交换。

### 顾客 (小程序端)
是系统的核心服务对象。他们向系统发送登录授权请求、商家与菜品浏览查询、购物车操作请求、订单提交与支付请求、退款/纠纷申请、评价提交以及地址管理请求。系统处理后，向顾客返回商家及菜品信息、订单状态实时更新（如：制作中、配送中、已送达）、骑手位置轨迹、支付结果、退款处理进度以及会员信息。

### 商家 (Web/小程序端)
是系统的商品与服务提供方。商家向系统发送入驻申请与登录请求、菜品与分类的全生命周期管理请求、订单处理指令（接单、出餐完成）、退款申请处理请求以及经营数据查询请求。系统处理后，向商家返回审核结果、管理操作反馈、新订单实时通知、销售与订单统计报表。

### 骑手 (小程序端)
是系统的物流执行方。骑手向系统发送实名认证与登录请求、上下线状态切换、接单/拒单请求、取餐与送达完成上报、实时GPS位置上报、配送异常情况上报以及收入提现请求。系统处理后，向骑手返回认证与登录结果、智能派单通知、订单导航信息、系统广播及收入明细。

### 系统管理员 (Web后台)
是平台的运营监管角色。管理员向系统发送登录请求、对商家与骑手入驻申请进行审核的指令、全局营销策略（优惠券）配置请求、系统参数配置请求、全平台订单监控与干预指令以及平台级数据报表查询请求。系统处理后，向管理员返回登录结果、待审核清单、实时监控仪表盘、平台经营综合分析报表。

### 微信支付平台
作为资金结算的外部渠道。系统向其发送预支付订单生成请求；支付平台处理完成后，向系统异步发送支付结果回调通知（成功/失败），触发系统更新内部订单状态并驱动后续业务流程。

## 成员 A — 后端开发（占比 40%）

### 负责模块
- `api/` — Spring Boot 应用层（端口 8080）
- `common/` — 共享模块
- `service/` — 业务逻辑层
- `init.sql` — 数据库初始化脚本（20 张表 + 种子数据）
- `docker-compose.yml` — Docker 环境配置
- `*.bat` / `*.ps1` — 启动/停止脚本
- `pom.xml` — Maven 依赖管理

### 主要职责

#### 1. 数据库设计与建模
- 设计 20 张数据库表（用户、商家、骑手、订单、菜品、评价、优惠券、配送轨迹等）
- 编写 `init.sql` 初始化脚本（建库、建表、种子数据）
- 订单状态机设计（待支付→已支付→备货中→已接单→配送中→已完成 / 退款流程）

#### 2. 公共模块（common）
- **20 个实体类**：User, Merchant, Rider, Order, Dish, Category, Review, Coupon, Payment, DeliveryTrack, DeliveryException, JointDeliveryGroup, JointDeliveryMember, OrderDetail, OrderDispute, RiderIncome, RiderWithdrawal, SystemConfig, UserAddress, UserCoupon
- **13 个 DTO**：登录、下单、骑手注册、纠纷创建、联合配送、评价提交等数据传输对象
- **8 个 VO**：订单、购物车、评价、骑手/商家附近搜索等视图对象
- **8 个状态常量**：订单状态、骑手状态、纠纷状态、配送异常状态等枚举
- **工具类**：JWT 令牌（access + refresh token）、Snowflake 雪花 ID 生成器、高德地图坐标转换/距离计算、身份证校验、ThreadLocal 用户上下文
- **统一封装**：Result 统一响应格式、BusinessException 业务异常、WebSocket 服务端

#### 3. 业务逻辑层（service）
- **25 个服务接口 + 16 个实现类**：用户、商家、菜品、分类、购物车、订单、支付、骑手、评价、优惠券、纠纷、联合配送、推荐、报表、系统配置、库存、配送调度、ETA 预估、地图定位、收入等级等
- **20 个 MyBatis-Plus Mapper**：对应全部数据表的持久层
- **RabbitMQ 延迟队列**：订单 30 分钟超时未支付自动取消 + 库存回滚
- **WebSocket 推送**：骑手位置实时推送、新订单通知

#### 4. 应用层（api）
- **15 个 REST 控制器**：认证、用户、商家、菜品、分类、购物车、订单、骑手、评价、优惠券、纠纷、联合配送、推荐、管理后台、系统配置
- **配置类**：WebMvc 静态资源映射与拦截器注册、Redis 配置（GEO + Lua 脚本）、RestTemplate、WebSocket、MyBatis-Plus 自动填充
- **拦截器**：JWT 登录拦截器、API 限流拦截器
- **过滤器**：XSS 防护过滤器
- **全局异常处理**：BusinessException / 405 / 500 统一处理

#### 5. 关键业务特性实现
- 智能派单（Redis GEO 搜索附近骑手 + 距离/负载加权评分）
- 联合配送（大订单拆分子任务，多骑手协作）
- Redis Lua 脚本原子性库存扣减/回滚
- RabbitMQ 延迟队列超时取消
- 退款流程（客户申请→商户同意/拒绝→回滚库存→标记支付已退款）
- 骑手异常上报（联系不上顾客、地址错误、商品损坏）
- DeepSeek API AI 菜品推荐
- 高德地图坐标转换与距离计算

#### 6. 环境与部署
- Docker Compose 编排（MySQL + Redis + RabbitMQ）
- Maven 多模块构建配置
- application.yml 主配置文件（数据库/Redis/RabbitMQ/JWT/高德地图/DeepSeek）
- 启动脚本（start-all.bat, start-docker-env.bat, stop-docker-env.bat）

## 成员 B — 前端开发（占比 30%）

### 负责模块
- `web-admin/` — 管理后台（Vue3 + Element Plus + ECharts 6，端口 5173）
- `web-customer/` — 客户端（Vue3 + Vant4 + 高德地图，端口 5174）

### 主要职责

#### 1. 管理后台 web-admin
- **仪表盘/数据大屏**：核心业务数据可视化（订单量、营收、骑手活跃数等 ECharts 图表）
- **商家审核管理**：商家入驻审核（查看营业执照、通过/驳回）
- **订单监控**：全平台订单实时监控、订单详情查看、异常订单处理
- **骑手管理**：骑手实名认证审核、骑手列表、骑手状态监控
- **用户管理**：用户列表、用户状态管理
- **优惠券管理**：创建/编辑/发布优惠券模板（满减/折扣/免配送费）
- **纠纷处理**：平台介入审核纠纷，处理退款申诉
- **系统配置**：系统参数键值对配置管理
- **报表导出**：商户报表、收入报表的查看与导出
- 路由守卫、登录认证、Token 管理、权限控制

#### 2. 客户端 web-customer
- **用户登录/注册**：微信登录对接、手机号绑定
- **首页/附近商家**：基于地理位置搜索附近商家、商家列表展示
- **商家详情页**：菜品分类浏览、菜品详情、加入购物车
- **购物车**：多商家购物车管理、菜品数量调整、总价计算
- **下单与支付**：选择收货地址、使用优惠券、提交订单、模拟支付
- **订单跟踪**：订单列表、订单状态实时跟踪（WebSocket 推送）、配送轨迹查看
- **评价系统**：对商家和骑手分别评价打分
- **个人中心**：收货地址管理、优惠券管理、订单历史
- **AI 推荐**：DeepSeek 智能菜品推荐展示
- 响应式布局适配移动端、Vant4 组件库集成、高德地图定位与距离显示

## 成员 C — 前端开发（占比 30%）

### 负责模块
- `web-merchant/` — 商户端（Vue3 + Vant4，端口 5176）
- `web-rider/` — 骑手端（Vue3 + Vant4 + 高德地图，端口 5175）

### 主要职责

#### 1. 商户端 web-merchant
- **商家入驻申请**：填写店铺信息、上传营业执照、提交审核
- **商家信息管理**：编辑店铺信息、查看审核状态、查看评分与评价
- **菜品管理**：菜品分类创建/编辑、菜品 CRUD（名称/价格/库存/图片/上下架）
- **图片上传**：菜品图片上传功能
- **订单管理**：新订单实时通知（WebSocket）、接单/拒单、备货完成通知
- **订单状态流转**：待处理→已接单→备货中→等待骑手取餐
- **退款处理**：查看退款申请、同意/拒绝退款
- **评价查看**：查看顾客对店铺的评价
- **收入报表**：营业额统计、订单量统计、收入趋势图表
- **实时通知**：新订单、退款申请等实时推送
- Vant4 移动端适配、Pinia 状态管理、Axios 请求封装

#### 2. 骑手端 web-rider
- **骑手认证**：实名认证（姓名、身份证号）、提交审核
- **上下线管理**：骑手上下线状态切换
- **接单大厅**：待分配订单列表、自动派单接收、抢单（如有）
- **订单取餐**：导航到商家取餐（高德地图导航）、确认取餐
- **订单配送**：导航到顾客地址（高德地图路线规划）、确认送达
- **GPS 轨迹上报**：实时上传骑手位置（WebSocket 推送）、轨迹记录
- **联合配送**：多骑手协作配送任务查看与参与
- **异常上报**：配送异常情况上报（联系不上顾客、地址错误、商品损坏等）
- **收入管理**：收入记录查看、收入明细、余额查询
- **提现**：骑手余额提现申请
- **等级与积分**：骑手等级查看、积分记录
- **订单历史**：已完成配送订单列表
- 高德地图 SDK 集成（定位、导航、轨迹显示）、WebSocket 实时通信、Vant4 组件

## 模块对接关系

```
成员 A (后端 40%)
    │
    │  提供 REST API (15 个 Controller)
    │  提供 WebSocket 推送
    │
    ├──────────────────────────────────────────
    │                    │                    │
    ▼                    ▼                    ▼
成员 B (前端 30%)   成员 C (前端 30%)   成员 C (前端 30%)
web-admin            web-customer         web-merchant + web-rider
(管理后台)            (客户端)             (商户端 + 骑手端)
```

### 接口对接要点
| 对接场景 | 后端接口 | 前端消费者 |
|----------|---------|------------|
| 用户认证 | `/api/auth/*` | B（客户）+ C（商户、骑手）|
| 商家/菜品 | `/api/merchant/*`, `/api/merchant/{id}/dishes/*` | A（后端核心）+ B（客户浏览）+ C（商户管理）|
| 订单全流程 | `/api/order/*` | B（客户下单）+ C（商户接单、骑手配送）|
| 骑手配送 | `/api/rider/*` | C（骑手端核心）|
| 管理后台 | `/api/admin/*` | B（管理后台核心）|
| WebSocket 推送 | WebSocket Server | B（客户订单状态）+ C（商户新订单、骑手派单）|
